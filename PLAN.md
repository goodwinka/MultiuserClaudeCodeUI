# План: Многопользовательский ClaudeCodeUI сервер

Цель: развернуть [ClaudeCodeUI](https://github.com/siteboon/claudecodeui) как многопользовательский сервис с изоляцией сессий, общей конфигурацией Claude Code и полной работой в оффлайн-режиме.

---

## 1. Архитектура

**Схема: Reverse Proxy → Auth Gateway → Process Manager → ClaudeCodeUI instances**

Один контейнер (или выделенный сервер), внутри которого:
- **Nginx** — reverse proxy, раздаёт статику, проксирует WebSocket
- **Auth Gateway** (Node.js/Express) — регистрация, логин, JWT, управление
- **Process Manager** — запускает/останавливает процессы claudecodeui для каждого пользователя
- **ClaudeCodeUI** — по одному процессу на активного пользователя, каждый на своём порту

---

## 2. Изоляция пользователей

- Каждый зарегистрированный пользователь → системный пользователь Linux (uid автоинкремент)
- Домашняя папка: `/data/users/{username}/projects/` — рабочая директория
- ClaudeCodeUI процесс запускается от имени этого системного пользователя (`su - username`)
- Изоляция обеспечивается правами файловой системы + запуском процесса из нужного cwd

---

## 3. Общая конфигурация Claude Code

- Глобальный `/etc/claude/settings.json` — шаблон настроек (эндпоинт локальной нейросети, MCP, параметры)
- При создании пользователя: `~/.claude` указывает на общие настройки (symlink на readonly часть)
- Переменные окружения (API URL, модель) задаются глобально в process manager

---

## 4. Компоненты и стек

| Компонент | Технология | Назначение |
|---|---|---|
| Reverse proxy | Nginx | Маршрутизация, SSL, WebSocket |
| Auth Gateway | Node.js (Express) | Регистрация, JWT, управление |
| БД пользователей | SQLite | Логины, хеши паролей, метаданные |
| Process Manager | Node.js демон | Запуск/остановка claudecodeui per-user |
| ClaudeCodeUI | siteboon/claudecodeui | UI для каждого пользователя |
| Claude Code | CLI | Бэкенд, настроен на локальную LLM |
| Инструменты | clangd, compilers, MCP server | Установлены глобально один раз |

---

## 5. Регистрация и аутентификация

1. Страница `/register` — поля: username, password. Без email, без подтверждения администратора
2. Валидация: username — `[a-z0-9_]`, длина 3–20; пароль — минимум 6 символов
3. При регистрации: запись в SQLite + системный пользователь Linux + папка проектов
4. Логин → JWT токен в httpOnly cookie
5. Middleware проверяет токен → определяет пользователя → проксирует к его процессу

---

## 6. Жизненный цикл сессии

```
Логин → Auth Gateway проверяет JWT
  → Process Manager: есть ли процесс для user?
    → Нет: запустить claudecodeui на порту N от имени user
    → Да: вернуть порт
  → Nginx проксирует /session/{user}/* → localhost:N
  → Таймаут бездействия (30 мин) → убить процесс
```

- Порты: пул 10000–11000, по одному на активного пользователя
- Максимум одновременных сессий — настраивается администратором

---

## 7. Маршрутизация Nginx

```
/                   → Auth Gateway (логин/регистрация)
/admin/*            → Admin Panel
/session/*          → Proxy к claudecodeui процессу пользователя
/session/*/ws       → WebSocket proxy
```

---

## 8. Структура файловой системы

```
/opt/claudecodeui/          ← клон репозитория (общий, readonly)
/etc/claude/                ← глобальные настройки claude code
/data/users/{username}/
  ├── projects/             ← рабочая директория (монтируется из хоста)
  └── .claude-session/      ← данные сессии пользователя
/var/lib/multiuser-ccui/
  ├── users.db              ← SQLite
  ├── auth-gateway/         ← код gateway
  └── logs/                 ← логи по пользователям
```

---

## 9. Docker

- Один `Dockerfile`: Ubuntu + Node.js + все компиляторы + clangd + MCP + Claude Code + ClaudeCodeUI
- `docker-compose.yml`:
  - Volume: `/data/users` → хостовая папка (проекты вынесены из контейнера)
  - Volume: `/var/lib/multiuser-ccui` → хостовая папка (БД, логи)
  - Volume: `/etc/claude` → хостовая папка (настройки)
  - Порт: один (80/443), всё через Nginx внутри контейнера

---

## 10. Админ-панель

Эндпоинт `/admin` (отдельный JWT с ролью `admin`). Возможности:
- Список пользователей, блокировка, удаление
- Просмотр активных сессий, принудительное завершение
- Просмотр логов (по пользователям)
- Изменение глобальных настроек Claude Code
- Мониторинг ресурсов (CPU/RAM на процесс)

---

## 11. Оффлайн-режим

- Все npm-зависимости установлены в образе заранее (нет внешних CDN)
- Claude Code настроен на локальный LLM-сервер (Ollama / vLLM / llama.cpp)
- Аутентификация — локальная SQLite, без внешних OAuth провайдеров
- Статика claudecodeui раздаётся локально через Nginx

---

## 12. Порядок реализации

| Этап | Содержание |
|------|-----------|
| 1 | **Инфраструктура**: Dockerfile с зависимостями, установка claudecodeui, claude code, компиляторов, clangd, MCP |
| 2 | **Auth Gateway**: регистрация, логин, SQLite, JWT, создание системных пользователей |
| 3 | **Process Manager**: запуск/остановка claudecodeui per-user, пул портов, таймауты бездействия |
| 4 | **Nginx конфиг**: проксирование HTTP + WebSocket, маршрутизация по сессиям |
| 5 | **Общий Claude Code конфиг**: шаблон settings.json, подключение к локальной LLM |
| 6 | **Админ-панель**: CRUD пользователей, мониторинг, управление сессиями |
| 7 | **Docker Compose**: volumes, сеть, env, автозапуск |
| 8 | **Тестирование и хардинг**: безопасность, лимиты ресурсов, логирование |
